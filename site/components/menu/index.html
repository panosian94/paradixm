--- title: Menu layout: component ---

A list of commands or options that do not need to be displayed at all times. These commands and options are organized for the user's ease of access.

Typical Appearance

Menus are most easily accessed through toolbars on the top of the screen for most devices. Toolbars usually have general labels that hide away similar preferences for a user to access once these labels are clicked on. Menus can also be hidden and accessed by right clicking a mouse. This drops down a (usually more limited) list of options to choose from.

Typical Behavior

Menus are organized into different labelled sections. When a menu is opened, it usually lists several different options that have to do with the access button label. For example, a common access button label across platforms is "File". When the "File" menu is opened, common options include "Save as..", "Save..", or "Open..". All these options have to do with behaviours that affect the File that is opened. Most menu options lead to a different screen that help users personalize settings specifically to their needs.

The most relevant menu event is the onClick. A onClick event indicates that the user has triggered the button.

Variants

  • Buttons are similar to menus. When a button is triggered, it can also open up new screens that contain options. However, buttons are not as broad as menus. Menus provide the user with organized information that allows them to change settings where buttons are just triggers that help save settings.

    • Priority Metrics

    Menus are used on almost every screen that users come into contact with. Because of this fact, menus should be easy to understand and learn. Labels should be clear and concise so that memorability of grouped commands and options make sense to the user when they are searching for something specific.

    The user's efficiency with the menu should be one of the most important priorities because the more complex a menu becomes, (such as hierarchical menus that use hovering) the more difficulty a user will have. The menu can become prone to errors in this way if the user is not easily adapted to the organization or setup of a menu. Satisfaction from users will be tied to the amount of time it takes them to complete their tasks.
